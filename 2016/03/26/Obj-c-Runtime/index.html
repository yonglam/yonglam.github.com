Obj-c Runtime

2016-03-26

Ojb-c是一门非常灵活的语言，是一门十足的动态语言，你几乎可以在运行时做任何编译时做的工作，包括创建类，定义方法，定义成员变量等等。而这些能力是通过runtime运行时提供的。下面我们简单介绍一下这个体系。

一、面向对象

1.对象

其实就是一块内存，内存中是各个类属性（property）的排列。这些类属性中第一个永远是一样的，叫“isa”，它是一个指针，指向它所属的类的内存。

2.类

每一个类，也是一块内存，结构和对象是一样的，它其实也是“元类”的对象。只不过，类自身除了“isa”指针外，还有一系列的属性，包括：一个数组指针指向所有的成员方法，一个数组指针指向所有的类方法，一个指针指向其父类等等。

3.元类

所有类的“isa”指针都指向它。

二、方法

1.基本概念

先让我们弄清楚三个概念：

  1. Method：由“方法名”、参数、代码块一起组成的整体。运行时使用“IMP”来表达。

  2. Message：由“方法名”、参数的组合，它会被发送给一个指定的对象。

  3. Selector：就是上述两者的“方法名”的表达。

  4. Message send：为Message找到正确的Method的过程。
    2.方法调用

对一个对象调用方法的时候：

编译器会转换成c函数的调用：objc_msgSend(obj, @selector(doSomething))。
运行时会通过obj的isa指针找到其所属的类，再在类的方法列表中查找doSomething这个方法。如果找到了就执行相应的方法指针指向的函数。否则会使用super指针向其父类的方向寻找。如果最后也没有找到那就会启用“方法转发”的机制。细心的你可能会发现如果有大量的方法调用，那上述的方法搜索会很耗时。所以每个类中有一个成员属性叫“cache”，用来缓存每次找到的方法。每次方法搜索都会先在cache中寻找。

3.方法的转发

每一个方法都是使用selector来唯一标识的。当向一个对象发送一个Message时，如果在对象的类的链条上找不到这个消息的selector所对应的方法的实现时，就会启用消息的转发机制。方法的转发有三种机制：

  1. 惰性加载

resolveInstanceMethod:，运行时会首先调用这个方法，让类本身有机会去实现这个方法。所以开发者可以重载这个方法，动态注入方法的实现。

  1. 快速转发

forwardingTargetForSelector:，其次会调用这个方法，询问是否有新执行者，让其执行这个方法。开发者重载这个方法，返回能够响应的对象。

  1. 标准转发

如果上述两个机制都没有解决方法的调用，那么运行时会调用methodSignatureForSelector:，来获取合适的方法签名，再调用forwardInvocation:来执行这个方法。

三、NSProxy代理

OC中没有多重继承，为了实现我们可以使用NSProxy来做到。NSProxy还可以达到延后创建对象等目的。
